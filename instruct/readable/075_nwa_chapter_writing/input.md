# 系统设定

你是一个网文高手，擅长运用X-Y轴配方系统帮助用户完成小说创作，从灵感明晰到最终成稿。

---

## 创作阶段说明

创作流程分为以下阶段，本次任务将指定你完成其中某一阶段：

### 阶段1：理解用户需求

判断用户灵感清晰度，明确创作方向（题材、情感倾向、篇幅范围）。

### 阶段2：生成配方方案

基于用户需求匹配X-Y轴配方，输出创作意图文档（JSON格式）。

**要求**：
- 匹配X轴剧情模式（36种之一）和Y轴标签（12种中选2-3个）
- 判断化学反应强度（↗/↘/✷）
- 生成配方方案

**验证要点**：
- X轴模式ID匹配 `^[A-G]\d{1,2}$` 格式（如A3, B12, G36）
- Y轴标签在12种枚举中，数量2-3个
- 化学反应强度在 [↗, ↘, ✷] 中
- 情感基调约束：sweet用↗、angsty用↘、suspense用✷

### 阶段3：写作准备

设计人物体系、人物关系和三层大纲结构（JSON格式）。

**人物设计**：
- 主角3-5个特质（含显性/隐性动机、秘密恐惧）
- 配角需明确功能定位：助力/阻碍/反转点
- 所有角色必须有具体名字或固定称谓

**人物关系设计**：
- 主要角色间的初始关系、核心张力/冲突点、关系变化轨迹
- 人物关系的转折点应与大纲转折点呼应

**大纲设计（三层结构）**：
- 第一层：每幕的story_synopsis（500-1000字故事梗概）
- 第二层：每章的chapter_synopsis（200-400字章节梗概）
- 第三层：每章的scene_sequence（至少3个场景，每个场景需8个必需字段：scene_number、scene_location、characters_present、scene_purpose、core_action、conflict_point、emotional_shift、scene_outcome）
- core_action必须是可视化具体行为，不能写"讨论"、"了解情况"等抽象表述
- 场景间必须有因果关系，忠实服务于chapter_synopsis
- 大纲必须有转折点（act_one/act_two的turning_point）
- 所有设计的人物必须出现在大纲中
- 务必在大纲设计阶段充分规划情节，确保情节可以撑起目标篇幅

### 阶段4：章节创作

按大纲逐章创作小说正文（Markdown格式）。

**要求**：
- 按大纲逐章创作，chapter必须是正文叙事，不可以是大纲等简略内容
- 用户给定了具体篇幅要求时，必须确保实际创作内容达到该字数要求，但切记不可注水填充字数
- 如果接近字数上限但情节未完，优先保证情节完整性
- **绝对禁止触碰用户的避雷红线**

---

## 内容质量底线

- **主题一致**：核心主题必须贯穿始终
- **角色一致**：主角不能前后矛盾或突然消失；性格、能力不能无故崩坏（除非有合理铺垫）
- **设计遵循**：角色实际表现必须符合角色设定中的设计（traits、motivation等）
- **逻辑自洽**：不能有前后矛盾（如死人复活、判决矛盾等）
- **情节密度**：章节必须包含足够的具体故事事件，避免纯对话或纯内心独白
- **情感交付匹配**：实际内容倾向必须与reaction_strength一致
- **语言纯净**：避免不合理的多语言混用（情节必需的人名、品牌等除外）
- **人物完整出场**：设计过的人物必须出现在大纲和正文中
- **正文叙事**：章节内必须是小说正文叙事，不可以是大纲性质的简略叙事



# 用户需求

写一个都市无限流的小说。主角是个只想躺平的"苟道中人"，性格比较滑稽有趣，特别怕死。千万不要那种无脑爽文或者系统流，主角没有强力金手指，全靠脑子、观察细节和利用规则漏洞来通关。剧情上要是那种他想躲但躲不掉，被迫卷入反派的阴谋，从一个小意外慢慢揭开背后某种宏大的宿命感。人物要有成长弧光，但不要套路化，主角风格要轻松搞笑一点，用来调节紧张的氛围，但逻辑一定要严密。10万字篇幅。

# 设定一致性管理指南

> 本文档指导如何在长篇创作中维护设定的一致性，避免前后矛盾的逻辑漏洞。
> 
> **背景**：在评测中发现，即使是顶级模型（Claude Opus 4.5、ERNIE 5.0、Gemini 3 Pro）在长篇创作时都会出现设定矛盾问题。这不是偶然失误，而是LLM长文本创作的共性短板。本指南提供系统性的防护机制。

---

## 为什么需要设定一致性管理

看三个真实的逻辑漏洞案例：

### 案例1：死亡年份矛盾（Claude Opus 4.5）

```
第2章（档案描述）：
"死亡日期：2022年3月15日...许言彻，已故"

第16章（墓碑描述）：
"许言彻之墓...2021年3月15日"
```

**问题**：同一个人的死亡年份，档案写2022年，墓碑写2021年。读者会困惑：到底哪年死的？

### 案例2：死亡时间矛盾（ERNIE 5.0）

```
第1章（开篇设定）：
"今晚23:00杀害你"

第11章（对话回忆）：
"她7点的时候还在整理档案"（暗示死亡发生在7点）
```

**问题**：23:00和7点相差16小时，完全无法自洽。

### 案例3：倒计时含义矛盾（Gemini 3 Pro）

```
第1章（能力描述）：
"距离杀害你：6天"（含义：苏清禾将杀陆沉）

第4章（能力描述）：
"距离被捕：XX天"（含义：显示苏清禾被捕倒计时）
```

**问题**：同一个能力，从"预示他人的行为"变成了"预示他人的命运"，核心机制被篡改。

---

## 设定登记簿机制

**核心思想**：在创作过程中，维护一个随时更新的"设定登记簿"，记录所有需要保持一致的关键信息。

### 登记簿模板

在开始创作后，创建并持续维护以下登记簿（可存储在 `writing_log.md` 中）：

```markdown
## 设定登记簿

### 1. 时间线锚点

| 事件 | 精确时间 | 首次出现章节 | 备注 |
|------|---------|-------------|------|
| 许言彻死亡 | 2022年3月15日 23:00 | 第1章 | 档案记录 |
| 故事开始 | 2024年10月1日 | 第1章 | 现在时间 |
| 苏清禾入职警局 | 2023年6月 | 第3章 | 背景设定 |

### 2. 超自然能力/世界观规则

| 能力名称 | 具体效果 | 限制条件 | 首次出现章节 |
|---------|---------|---------|-------------|
| 死亡倒计时 | 看到他人头顶显示死亡时间 | 只能看见，无法改变 | 第1章 |
| 倒计时显示内容 | "距离XX：N天/小时" | XX只能是"死亡"或"杀害" | 第1章 |

### 3. 角色关键属性

| 角色 | 关键属性 | 当前状态 | 首次出现章节 |
|------|---------|---------|-------------|
| 陆沉 | 左手无名指旧伤疤 | 存在 | 第2章 |
| 苏清禾 | 锁骨下方胎记 | 存在 | 第5章 |
| 苏清禾 | 对猫毛过敏 | 是 | 第3章 |

### 4. 物件状态追踪

| 物件 | 当前状态 | 位置 | 最后更新章节 |
|------|---------|------|-------------|
| 凶器手术刀 | 完整 | 苏清禾包中夹层 | 第4章 |
| 陆沉的手机 | 已安装窃听软件 | 陆沉持有 | 第6章 |
| 证据U盘 | 已损坏 | 警局证物室 | 第8章 |

### 5. 数值倒计时追踪

| 倒计时名称 | 当前值 | 对应故事时间 | 最后更新章节 |
|-----------|-------|-------------|-------------|
| 杀害倒计时 | 6天 | 第1天 | 第1章 |
| 杀害倒计时 | 5天 | 第2天 | 第3章 |
| 杀害倒计时 | 3天 | 第4天 | 第7章 |
```

### 登记簿维护规则

1. **新设定必登记**：任何新出现的时间、数值、能力规则、角色属性，必须在创作后立即登记
2. **使用前必查询**：提及已有设定时，必须先查询登记簿确认当前值
3. **变化必更新**：任何设定的变化（物件损坏、倒计时减少等）必须更新登记簿
4. **冲突必标记**：如果发现查询结果与计划写法冲突，优先修改计划写法

## 章节前一致性检查

在开始写作新章节之前，执行以下检查流程：

### 检查清单

```
BEFORE_CHAPTER_CHECKLIST:

□ 1. 时间连续性
   - 上一章结束的时间点是？_______________
   - 本章开始的时间点是？_______________
   - 如果有时间跳跃，倒计时是否同步更新？

□ 2. 空间合理性
   - 上一章结束时各角色的位置？
     - 主角A：_______________
     - 主角B：_______________
   - 本章开始时各角色的位置？
   - 位置转移是否合理（有足够时间/交通方式）？

□ 3. 状态延续性
   - 上一章有角色受伤吗？伤势应该如何？
   - 上一章有物件被使用/损坏吗？状态应该如何？
   - 上一章有情绪爆发吗？余波应该如何？

□ 4. 信息一致性
   - 本章需要提及哪些已有设定？
   - 已查询登记簿确认这些设定的正确值？

□ 5. 倒计时同步
   - 故事内时间过了多久？
   - 各倒计时数值应该减少多少？
```

### 检查示例

假设要写第7章：

```
上一章（第6章）状态回顾：
- 结束时间：2024年10月4日晚上
- 陆沉位置：自家公寓
- 苏清禾位置：自家公寓
- 杀害倒计时：应该是3天（10月1日是6天，过了3天）
- 陆沉状态：右肩被玻璃划伤，已简单包扎

第7章开始时应该：
- 时间：10月5日早上（或4日深夜）
- 陆沉的肩伤：还在，应该有疼痛感或行动不便
- 倒计时：如果是10月5日，应该显示2天
```

## 章节后自检步骤

在完成每章创作后，执行以下自检：

### 自检流程

```
AFTER_CHAPTER_CHECKLIST:

□ 1. 新设定登记
   - 本章是否引入了新的时间点？→ 登记
   - 本章是否引入了新的数值/规则？→ 登记
   - 本章是否引入了新的角色属性？→ 登记
   - 本章是否有物件状态变化？→ 更新登记

□ 2. 一致性扫描
   - 全文搜索本章中的所有数字/日期
   - 逐一与登记簿比对：
     - 日期格式一致？（2022 vs 22年）
     - 数值准确？（倒计时、年龄、时间）
     - 名称一致？（角色名、地名）

□ 3. 能力/规则使用检查
   - 本章中超自然能力的使用是否符合已定义的规则？
   - 是否无意中扩展或修改了能力的边界？

□ 4. 跨章节一致性
   - 如果本章提及了之前章节的事件，描述是否一致？
   - 角色对过去事件的回忆是否与原文吻合？
```

### 自检示例

完成第16章后的自检：

```
本章扫描结果：
- 出现日期："2021年3月15日"（墓碑）

与登记簿比对：
- 登记簿记录：许言彻死亡日期 = 2022年3月15日

→ 发现矛盾！
→ 修正方案：将墓碑日期改为"2022年3月15日"
```

## 高频漏洞点提醒

基于评测案例分析，以下是最容易出错的设定类型：

### 1. 年份/日期（最高危）

**典型错误**：
- 2022年 vs 2021年
- 3月15日 vs 3月5日
- "三年前" vs "两年前"

**防护措施**：
- 所有年份首次出现时立即登记
- 使用相对时间（"三年前"）时，同时计算并登记绝对时间
- 墓碑、档案、证件等正式文件的日期必须与登记簿核对

### 2. 倒计时/数值（高危）

**典型错误**：
- 倒计时减少速度与故事时间不匹配
- 倒计时含义在中途被篡改
- 百分比/金额/距离等数值前后不一

**防护措施**：
- 建立"倒计时追踪表"，每章更新
- 倒计时的含义（显示什么内容）在首章严格定义，后续章节只引用不重新定义
- 任何数值首次出现即登记

### 3. 超自然能力边界（高危）

**典型错误**：
- 能力从"只能看见"变成"可以干预"
- 能力的触发条件被无意放宽
- 能力的显示内容被无意修改

**防护措施**：
- 能力定义必须包含：效果、限制、触发条件、显示格式
- 后续章节使用能力时，严格引用首次定义
- 如需升级能力，必须有明确的剧情事件作为理由

### 4. 身体细节（中危）

**典型错误**：
- 伤口位置从左手变右手
- 胎记位置从锁骨变肩胛
- 疤痕在某章突然消失

**防护措施**：
- 身体特征（疤痕、胎记、伤口）首次描写时登记具体位置
- 后续提及时查询登记簿
- 伤口的愈合进度需要与故事时间匹配

### 5. 角色位置/在场状态（中危）

**典型错误**：
- 角色A明明去了外地，下一章突然出现在本地
- 角色B在某场景中说话，但前文没有描写他到场

**防护措施**：
- 每章结束时记录各主要角色的位置
- 角色出场前确认其上次位置和移动合理性
- 群戏场景明确列出在场角色名单

## WRITING_TECHNIQUE_GUIDE.md

# 写作技巧指南 - 实战创作技法参考

> 本文档提供小说章节创作的实战写作技巧，包含场景构建、对话、叙事、情感渲染、冲突设计、节奏控制、开篇、结局、人物刻画共9大类。用于阶段4（章节创作）时参考。

---

### 2.1 场景构建技巧

#### 2.1.1 五感描写法(让读者身临其境)

**原则**: 每个场景至少调动3种感官(视觉+听觉+触觉/嗅觉/味觉)

**反面示例**(干瘪无感):

```
林羽走进黑客基地,看到幽灵坐在电脑前。
```

**正面示例**(五感丰富):

```
林羽推开锈蚀的铁门,刺鼻的机油味和电路板烧焦的气味扑面而来(嗅觉)。地下室只有显示器发出的幽蓝冷光,照得墙上的影子扭曲颤动(视觉)。键盘的敲击声密集如雨点(听觉),夹杂着服务器低沉的嗡鸣(听觉)。幽灵头也不回,手指在机械键盘上飞舞,每一下按键都带着清脆的咔哒声(触觉暗示+听觉)。
```

**技巧要点**:

- **视觉**: 光影对比、色彩冷暖、空间布局
- **听觉**: 环境音(背景)+特殊音(突出事件)
- **触觉**: 温度、质感、痛感
- **嗅觉**: 情绪记忆触发器(烟味/花香/血腥味)
- **味觉**: 内心状态外化(苦涩/甘甜)

#### 2.1.2 氛围营造三要素

**1. 光影控制情绪**

- 明亮温暖 → 希望/安全/温馨
- 昏暗阴冷 → 压抑/危险/绝望
- 闪烁不定 → 不安/混乱/惊悚

**2. 空间布局暗示关系**

- 开阔空间 → 自由/孤独/渺小
- 狭窄空间 → 压迫/紧张/亲密
- 角色位置 → 权力关系(高低/远近/正侧)

**示例**(用空间暗示权力关系):

```
总监张伟站在会议桌的顶端,背靠落地窗,逆光让他的脸完全陷入阴影。林羽坐在长桌另一头,所有人的目光都像聚光灯一样打在他身上。他感到自己像被钉在审判席上的囚犯。
```

**3. 物件细节传递信息**

- 干净整洁 → 控制欲强/强迫症
- 杂乱无章 → 混乱心理/贫穷
- 特定物件 → 身份暗示(茶具→文人/刀具→杀手)

#### 2.1.3 时间压缩与扩张

**压缩原则**: 日常琐碎场景快速跳过(洗漱/通勤)
**扩张原则**: 关键冲突场景慢镜头细描(生死对决/告白)

**反面示例**(平铺直叙无重点):

```
林羽早上7点起床,洗漱后吃了早餐,8点出门坐地铁,9点到公司,打开电脑开始工作。11点接到一个电话,是总监叫他去会议室。他走进会议室,发现所有高管都在。
```

**正面示例**(压缩无关,扩张关键):

```
林羽照常到公司。(压缩早晨琐碎)

11点,手机震动。总监张伟的来电显示让他心里一紧。(扩张开始)

"林羽,会议室,现在。"对方挂断前语气冷得像冰渣。

他推开24楼会议室的门,空气仿佛瞬间凝固。技术部全员、法务、HR、甚至CEO都在——这个阵仗,不是升职,是……宣判。(扩张冲突时刻,每个细节都值得写)
```

### 2.2 对话写作技巧

#### 2.2.1 潜台词设计(说A指B)

**原则**: 高质量对话必须有**表层含义**和**深层含义**的错位

**反面示例**(直白无味):

```
"你爱我吗?"
"我爱你。"
"那我们在一起吧。"
```

**正面示例**(潜台词丰富):

```
"你会一直在吗?"(表层:询问,深层:担心被抛弃)
他沉默了三秒,视线落在她肩膀上:"我会的。"(表层:承诺,深层:没有看她眼睛=心虚)
"你看着我说。"她的声音开始颤抖。(表层:要求,深层:已经察觉谎言)
```

**潜台词类型**:

- **权力试探**: "你觉得呢?"(我想看你服从or反抗)
- **情感试探**: "你今天真好看。"(我想知道你在意我吗)
- **威胁暗示**: "希望你做正确的选择。"(否则后果自负)
- **拒绝缓冲**: "我再想想。"(答案是拒绝但不想直说)

#### 2.2.2 对话节奏控制

**快节奏**(紧张/冲突):

- 短句、断句、频繁插话
- 省略主语(直接动作)
- 无完整句式

**示例**:

```
"你背叛了我们!"
"我没有——"
"闭嘴!"张伟一拳砸在桌上,"证据都在这里,你还想狡辩?"
"那不是我——"
"保安,带他走。"
```

**慢节奏**(回忆/哲思):

- 长句、完整句式、从容叙述
- 留白、停顿、深呼吸

**示例**:

```
幽灵靠在椅背上,手指在键盘上轻轻敲击,像在弹奏一首古老的曲子。半晌,他开口:"你知道吗,这个世界上最恐怖的不是死亡……"他停顿,目光望向远方,"而是你发现,你一直以为的真相,从一开始就是谎言。"
```

#### 2.2.3 人物语言风格差异化

**原则**: 不同性格/身份的角色说话方式必须有明显区别

| 角色类型 | 语言特征                          | 示例                             |
| -------- | --------------------------------- | -------------------------------- |
| 理工男   | 逻辑严密、专业术语、冷静客观      | "根据数据推算,成功率不超过23%。" |
| 文艺女   | 比喻丰富、情感细腻、诗意表达      | "那感觉就像在雨后看见彩虹。"     |
| 霸道总裁 | 命令句式、简洁直接、不容置疑      | "把方案改了,明天之前交给我。"    |
| 小混混   | 口语化、脏话/语气词多、语法不规范 | "哥几个,整他丫的!"               |
| 老学究   | 文言色彩、引经据典、语速缓慢      | "古人云,祸从口出……"            |

### 2.3 叙事技巧

#### 2.3.1 视角控制(谁在讲故事?)

**第一人称**(我):

- 优点: 代入感强、内心独白自然
- 缺点: 只能写"我"看到的,信息受限
- 适用: 悬疑推理、心理剖析、成长故事

**第三人称全知**(上帝视角):

- 优点: 自由切换场景/人物、信息全面
- 缺点: 读者距离感强、难以深入内心
- 适用: 群像剧、多线叙事、史诗故事

**第三人称限知**(跟随某个角色):

- 优点: 兼具代入感和灵活性
- 缺点: 需要技巧平衡信息透露
- 适用: 网文最常用,推荐

**关键原则**: **同一场景内不要频繁切换视角**(会让读者混乱)

#### 2.3.2 叙事距离调节

**远距离**(宏观/概述):

```
那一年,林羽经历了人生最黑暗的三个月。他被公司陷害,失去了工作,女友离开,父母病重。
```

**近距离**(微观/细节):

```
林羽盯着手机屏幕上的分手短信,拇指悬在"删除"键上方,却始终按不下去。窗外的雨声砸在玻璃上,和他的心跳声重叠在一起,咚、咚、咚……
```

**使用原则**:

- 重要情节 → 近距离细描
- 过渡衔接 → 远距离概述
- 情感高潮 → 极近距离(慢镜头)

#### 2.3.3 时间处理技巧

**倒叙**: 从结果开篇,再讲述原因(制造悬念)
**插叙**: 中途插入回忆/背景信息(解释动机)
**平叙**: 多线并行,交替推进(如《权游》)

**倒叙示例**:

```
【开篇】林羽站在公司楼顶,脚尖已经悬空。下面是30层的虚空,和围观的人群。他闭上眼,松开了手……

【倒叙】三个月前,一切还很正常。那天早上,他照常去公司……
```

### 2.4 情感渲染技巧

#### 2.4.1 Show, Don't Tell(展示而非告知)

**Tell**(直接告诉,无感):

```
林羽非常愤怒。他很难过,心情糟糕透了。
```

**Show**(通过细节展示,有感):

```
林羽死死攥着拳头,指甲掐进掌心,却感觉不到痛。他的喉咙像被什么东西堵住,想喊却发不出声。咬紧的牙关咯吱作响,太阳穴的青筋一跳一跳……
```

**Show的三种方法**:

1. **生理反应**: 心跳加速、手心出汗、肌肉紧绷
2. **肢体动作**: 拳头紧握、眼神飘忽、步伐急促
3. **环境反应**: 周围声音变得遥远、时间流速感改变

#### 2.4.2 情感层次递进(不要一步到位)

**错误**(情感突变):

```
林羽看到女友和陌生男人在一起,瞬间崩溃大哭。
```

**正确**(情感阶梯):

```
林羽愣住,大脑一片空白。(第一层:震惊麻木)

他眨了眨眼,以为自己看错了。可那确实是她,她的头靠在那个男人肩上,笑容灿烂。(第二层:怀疑否认)

胸口像被什么东西狠狠撞击了一下,呼吸骤然变得困难。(第三层:生理反应)

他机械地转身,走了两步,腿软得几乎站不住。(第四层:逃避)

泪水不受控制地涌出来,模糊了视线。(第五层:崩溃)
```

**情感阶梯通用模型**:

1. **冲击** → 2. **否认** → 3. **生理反应** → 4. **情绪爆发** → 5. **接受/崩溃**

#### 2.4.3 对比反差放大情感

**原则**: 越是压抑的人,崩溃时越震撼

**示例**(冷静理智的人崩溃):

```
林羽一直是公认的"冰山",从不情绪化。但这一刻,他蹲在墙角,双手抱头,像个无助的孩子。那些看似坚不可摧的理智防线,在真相面前,轰然倒塌。
```

**对比类型**:

- 性格对比: 开朗的人突然沉默 / 严肃的人突然笑
- 处境对比: 昔日辉煌vs现在落魄
- 预期对比: 以为是好消息→实际是坏消息

### 2.5 冲突设计技巧

#### 2.5.1 外部冲突 vs 内部冲突

**外部冲突**(人vs人/环境/制度):

- 主角vs反派
- 主角vs自然灾害
- 主角vs社会规则

**内部冲突**(人vs自己):

- 道德两难(救一个人vs救一百人)
- 欲望vs理智(报仇vs原谅)
- 恐惧vs勇气(逃跑vs面对)

**高级技巧**: **外部冲突压力下,内部冲突才更有张力**

**示例**:

```
【外部冲突】林羽必须在24小时内找到证据,否则公司会销毁所有痕迹。

【内部冲突】但唯一的线索在他前女友手里,她背叛了他,他发誓再也不见她……

【冲突叠加】时间一分一秒流逝,他的手停在通讯录上方,拇指颤抖——打,还是不打?
```

#### 2.5.2 冲突升级阶梯

**原则**: 冲突要层层递进,每次升级都出乎读者意料

**三幕结构冲突升级**:

- **第一幕**: 小冲突(被诬陷/被追捕)
- **第二幕**: 中冲突(发现更大阴谋/盟友背叛)
- **第三幕**: 大冲突(生死对决/道德抉择)

**反面示例**(冲突无递进):

```
第一章:主角被诬陷
第二章:主角被追捕
第三章:主角找到证据
第四章:主角复仇成功
```

**正面示例**(冲突层层升级):

```
第一章:主角被诬陷(小冲突)
第二章:主角逃跑时发现陷害者是最信任的人(冲突升级+情感背叛)
第三章:主角找到证据,但公开会导致无辜者受牵连(道德两难)
第四章:主角复仇成功,但付出惨痛代价(胜利中有悲剧)
```

### 2.6 节奏控制技巧

#### 2.6.1 紧张-舒缓交替(张弛有度)

**原则**: 持续高压会让读者疲劳,必须给喘息空间

**节奏设计**:

```
【紧张】追逐战(1000字)
  ↓
【舒缓】安全屋休息,回忆过去(300字)
  ↓
【紧张】发现藏身处暴露(800字)
  ↓
【舒缓】盟友救援,短暂安全(200字)
  ↓
【高潮紧张】最终对决(1500字)
```

**技巧**:

- 紧张段落:短句+快节奏+密集动作
- 舒缓段落:长句+慢节奏+内心独白/回忆

#### 2.6.2 信息释放节奏

**原则**: 读者知道60%-70%,主角知道30%-40%,保持信息差

**反面示例**(一次性交代清楚,无悬念):

```
原来,总监张伟一直在贪污公款,林羽无意中发现了证据,所以张伟设计陷害他,想让他坐牢,然后销毁证据。而幽灵其实是林羽的大学同学,一直在暗中帮助他……
```

**正面示例**(分段释放,保持悬念):

```
第1章:林羽被陷害(读者和主角都不知道原因)
第3章:林羽发现异常资金流动(读者和主角开始怀疑)
第5章:幽灵暗示"你触碰了不该碰的东西"(读者猜测是贪污,主角还没意识到)
第7章:林羽看到财务报表,恍然大悟(主角追上读者认知)
第9章:幽灵身份揭晓(新信息,读者和主角同步震惊)
```

#### 2.6.3 场景长短搭配

**原则**:

- 关键情节场景 = 1500-3000字(细描)
- 过渡场景 = 300-800字(速写)
- 高潮场景 = 3000-5000字(慢镜头)

**反面示例**(所有场景等长,无重点):

```
第一场:会议室对质 (1200字)
第二场:停车场逃跑 (1200字)
第三场:黑客基地见面 (1200字)
第四场:调查线索 (1200字)
```

**正面示例**(长短搭配,有主次):

```
第一场:会议室对质 (2500字,重点细描,情感爆发)
第二场:停车场逃跑 (600字,快节奏过渡)
第三场:黑客基地见面 (1800字,新角色引入+信息点)
第四场:调查线索 (400字,压缩琐碎)
第五场:真相对决 (4000字,高潮慢镜头)
```

### 2.7 开篇技巧

#### 2.7.1 钩子设计(第一句话抓住读者)

**原则**: 开篇100-300字内必须有**冲突/悬念/反常**

**反面示例**(平淡无奇):

```
林羽是一名程序员,在一家互联网公司工作。他今年28岁,单身,性格内向。今天是周一,他像往常一样来到公司……
```

**正面示例**(立即制造悬念):

```
【类型1:冲突开篇】
林羽被两个保安架着走出会议室时,脑子里只有一个念头:我被设计了。

【类型2:反常开篇】
林羽盯着电脑屏幕上那行代码,手开始颤抖——这段代码,他从未写过,但它确实出现在他的提交记录里,时间戳显示:凌晨3点17分。而那时,他正在家里睡觉。

【类型3:悬念开篇】
如果那天早上,林羽没有打开那封邮件,后面的一切都不会发生。可惜,世界上没有如果。
```

**钩子类型**:

- 危机开篇(正在发生的冲突)
- 秘密暗示(隐藏的真相)
- 倒计时(24小时内必须完成)
- 反常细节(不对劲的地方)

#### 2.7.2 世界观建立(不要信息轰炸)

**原则**: **通过角色行动展示世界规则,而不是大段说明**

**反面示例**(枯燥讲解):

```
2086年,赛博朋克时代,人类大部分肢体都可以替换为义体。大公司控制了政府,贫富差距极大,上层住在空中城市,下层住在地表贫民窟。规则怪谈开始出现,如果违反某些神秘规则,就会死亡……
```

**正面示例**(通过情节展示):

```
林羽擦了擦眼角——这是他身上唯一还是"真货"的部位。左臂的义体发出低沉的嗡鸣,提示电量只剩15%。

他抬头看了眼天空,浮空岛在云层之上闪烁着金色的光芒,那是上层阶级的乐园。而他脚下的地表贫民窟,永远只有昏暗的霓虹和永不停歇的酸雨。

墙上贴着红色警告:"晚上23点后禁止照镜子,违者后果自负。"旁边是一张失踪者海报,照片上的人半边脸扭曲变形,像融化的蜡像。

这就是2086,欢迎来到地狱。
```

### 2.8 结局技巧

#### 2.8.1 情感释放(完成情感承诺)

**原则**: 结局必须给读者期待的**情感答案**(爽/虐/升华)

**↗甜爽向结局**:

- 主角复仇成功,坏人得到报应
- 恋人终成眷属,HE结局
- 主角实力被认可,打脸成功

**↘虐心向结局**:

- 主角付出惨痛代价(爱人死亡/自己牺牲)
- 真相揭露但为时已晚
- 悲剧循环,无法打破

**✷烧脑向结局**:

- 多重反转,最后一个反转颠覆所有
- 开放式结局,留白让读者思考
- 真相即疯狂,主角/读者认知崩塌

#### 2.8.2 回扣前文(首尾呼应)

**技巧**: 结局中出现开篇的**关键意象/物件/台词**,制造圆满感

**示例**:

```
【开篇】林羽记得父亲临终前说的最后一句话:"记住,真相比你想象的更残酷。"那时他还是个孩子,不明白这句话的含义。

【结局】林羽站在真相面前,终于理解了父亲那句话。他闭上眼,泪水滑落——父亲,你是对的,真相比我想象的,残酷太多太多……

但至少,我知道了。
```

**回扣元素**:

- 开篇物件(手表/照片/信件)
- 开篇场景(同一地点再次出现)
- 开篇台词(重复或改写)

#### 2.8.3 余韵设计(最后一句话的力量)

**原则**: 最后100字应该有**情感冲击力**或**哲思留白**

**反面示例**(草草结束):

```
林羽终于复仇成功了。他回到家,洗了个澡,然后睡觉。第二天,他开始了新的生活。

——全文完——
```

**正面示例**(余韵悠长):

```
【类型1:物件回扣】
林羽把那张照片放回抽屉深处,锁上。

照片上,年轻的自己和父亲笑得很灿烂。那时他们还不知道,命运已经写好了剧本,而他们,不过是棋子。

但至少,这一局,他赢了。

【类型2:哲思留白】
很多年后,林羽偶尔会想起那个雨夜。

他不知道如果当时做了另一个选择,会不会有不同的结局。

但他知道,有些路,一旦走上,就再也回不了头。

【类型3:开放式】
林羽转身离开,消失在人群中。

没有人知道他去了哪里。

也许是新的城市,也许是……另一个故事的开始。
```

### 2.9 人物刻画技巧

#### 2.9.1 人物矛盾性(立体而非脸谱化)

**原则**: 人物必须有**性格矛盾点**,才不会像纸片人

**脸谱化角色**(扁平):

```
林羽:聪明、冷静、技术天才,没有缺点,永远正确。(完美人设=无聊)
```

**立体角色**(矛盾):

```
林羽:
- 表面:冷静理智,从不冲动
- 内核:内心孤独,渴望被理解
- 矛盾点:极度理性的人,一旦情感防线被突破,会比任何人都脆弱

【具体化】平时面对危险能冷静分析对策,但一旦有人提到他的父亲,他会瞬间失控。
```

**矛盾性类型**:

- 外在vs内在(表面强势/内心自卑)
- 过去vs现在(曾经善良/现在冷酷)
- 理智vs情感(大脑说该恨/心却放不下)

#### 2.9.2 人物成长弧(变化轨迹)

**原则**: 主角的**价值观/性格**必须在故事中发生改变

**无成长弧**(主角从头到尾一样):

```
第1章:林羽是个聪明冷静的程序员
第10章:林羽还是个聪明冷静的程序员,只是打败了敌人
```

**有成长弧**(主角蜕变):

```
第1章:林羽相信规则,相信公司,相信系统(天真)
第5章:林羽发现规则可以被操控,开始怀疑(幻灭)
第8章:林羽学会打破规则,变得像敌人一样冷酷(堕落边缘)
第10章:林羽选择坚守底线,用正义手段复仇(升华)
```

**成长弧关键点**:

- **信念动摇**: 原本坚信的东西被打破
- **试错挣扎**: 尝试新方法,但付出代价
- **顿悟时刻**: 找到真正的答案
- **行动证明**: 用行动体现新的价值观

#### 2.9.3 细节刻画(小动作/口头禅/习惯)

**原则**: 给每个重要角色设计**独特标签**,让读者一眼认出

**示例**:

```
林羽:紧张时会不自觉地推眼镜,即使没戴眼镜
幽灵:说话前总是先停顿三秒,像在搜索数据库
张伟:喝茶时小指会翘起,暴露精英阶层的傲慢
```

**标签类型**:

- 肢体习惯(咬指甲/转笔/挠头)
- 口头禅("嗯……怎么说呢""确实如此")
- 情绪反应(生气时耳根发红/尴尬时揉鼻子)

## 使用建议

**在创作chapters时**:
1. 应用**场景构建技巧**(五感描写+氛围营造)
2. 应用**对话写作技巧**(潜台词+节奏+人物语言差异)
3. 应用**情感渲染技巧**(Show Don't Tell+情感阶梯+对比反差)
4. 控制**节奏**(紧张-舒缓交替+信息释放节奏+场景长短搭配)
5. **开篇**必须有钩子(前300字制造冲突/悬念/反常)
6. **结局**完成情感承诺+回扣前文+余韵设计

**在创作全程**:
7. **人物刻画**保持矛盾性+成长弧+细节标签
8. **冲突设计**叠加外部冲突+内部冲突+层层升级
9. **超过8000字的长文**: 及时更新writing_log.md,下一章开始前先读log回顾

# 第一章 摸鱼达人的非自愿加班

晚上七点整，许昼安的手指精准地悬停在关机键上方。

他用余光扫了一眼四周——工位上的同事们正在收拾东西，茶水间传来有人倒水的声音，组长的位置是空的。

完美。

他默念三二一，正要按下那个代表自由的按钮，手机震了一下。

「昼安哥！部门聚餐！老地方川菜馆！」

是隔壁工位小陈发来的消息。许昼安的手指在键盘上悬停了零点三秒，然后若无其事地把手机屏幕朝下扣在桌上。

没看到。

他就是没看到。

一个合格的摸鱼达人必须掌握的核心技能之一，就是选择性失明。聚餐意味着社交，社交意味着消耗能量，消耗能量意味着明天的摸鱼质量会下降。这是一个恶性循环，必须从源头掐断。

许昼安心安理得地点击了关机。

屏幕变黑的瞬间，一只手拍上了他的肩膀。

"昼安，正好你还没走。"

许昼安脊背一僵。他认得这个声音——组长老王，四十五岁，秃顶，啤酒肚，擅长在下班时间出现并分配任务。

"王哥。"许昼安转过身，脸上挂着他练习了三年的标准化微笑，"有事？"

"有个紧急任务。"老王把一摞文件夹拍在他桌上，"上面催得急，一批敏感内容审核，今晚必须处理完。你不是审核速度最快的吗？交给你我放心。"

许昼安看着那摞至少有二十厘米厚的文件夹，笑容险些崩裂。

"王哥，我今天身体有点……"

"年轻人哪有那么娇气。"老王已经转身往门口走了，"加油，我看好你！"

门在他身后关上。

许昼安盯着那扇门，又看了看桌上的文件夹，最后把目光投向天花板。

"上辈子造了什么孽啊……"

他拖着椅子重新坐好，把手机捞起来瞄了一眼——小陈又发来好几条消息，还配了川菜馆的菜单照片。许昼安面无表情地把聊天界面上滑，假装没看见。

既然社交被强制取消了，那就……把这个不幸化作摸鱼的动力吧。

他重新开机，打开审核系统，开始机械地浏览那些等待处理的图片和文案。

说实话，内容审核是个很无聊的工作。每天对着屏幕看各种用户上传的东西，该删的删，该标记的标记，像流水线上的工人一样重复同样的动作。但许昼安觉得这份工作最大的优点就是——不用动脑子。

不用动脑子就意味着可以摸鱼。

他一边审核一边听歌，偶尔刷刷手机，效率低得理直气壮。反正老王也不知道这些东西到底要审多久，他说今晚干完就今晚干完，至于是今晚八点还是今晚十二点，那就是细节问题了。

时间一分一秒过去。

办公室的灯一盏接一盏熄灭，同事们陆续离开，最后只剩下许昼安工位的那一小片光亮。咖啡喝了三杯，外卖吃了一份，垃圾桶里堆满了零食袋子。他揉了揉酸涩的眼睛，看了看进度条——百分之七十八。

差不多了。

他打了个哈欠，手指继续机械地点击。下一张图片加载出来，是一张博物馆内景照片。

许昼安的手指顿了顿。

照片里是一座古典风格的博物馆大厅，穹顶很高，灯光昏暗，陈列着各种雕塑和油画。乍一看很普通，但许昼安盯着看了几秒后，眉头皱了起来。

有哪里不对。

他放大图片，视线在画面上来回扫描。雕塑，油画，展柜，玻璃罩……

玻璃罩。

他的目光定在画面右侧的一个玻璃展柜上。展柜里陈列的是一尊青铜人像，但展柜玻璃的反光里……

有一张脸。

一张模糊的、苍白的人脸，就那么直愣愣地贴在玻璃上，像是有人站在展柜内部往外看。

许昼安下意识想看看这张脸对应的位置——但按照展柜的角度和相机的拍摄方位来推算，那个位置不可能站人。除非……那个"人"就在展柜里面。

一阵凉意从后背窜上来。

他盯着那张模糊的脸看了好几秒，然后"啧"了一声，把图片关掉了。

"管他呢。"

他点了"通过"，继续审核下一张。

多一事不如少一事——这是摸鱼达人的第二条生存法则。就算那张图片真有问题，也不关他的事。上面的人想怎么处理是上面的事，他只是个打工的，工资又不会因为他多管闲事而变多。

但那张博物馆照片的画面还是在他脑海里留下了一点模糊的印象，像眼角余光瞥见的影子，挥之不去。

终于，进度条走到了百分之百。

许昼安长出一口气，关掉电脑，站起来活动了一下僵硬的脖子。手机屏幕亮起，时间显示——23:47。

"妈的。"

他骂骂咧咧地收拾东西，心想老王要是明天敢再给他加任务，他就……就忍着。

毕竟他还要还房租。

收拾完毕，许昼安没有立刻离开，而是往楼梯口走去。

公司在一栋写字楼的第18层，天台在顶楼可以自由出入。许昼安有个习惯，加班到深夜的时候就会去天台吹吹风，让脑子从工作模式里切换出来，否则回家也睡不着。

电梯到了顶楼，他推开那扇熟悉的铁门。

夜风灌进来，带着初秋的凉意。

天台不大，四周有一圈齐腰高的围栏，地上散落着几个被人遗弃的花盆和烟头。许昼安走到围栏边，掏出手机看了看——小陈的聚餐消息还挂在那儿，没有回复过。

"……"

他犹豫了一下，还是打了几个字发过去：「加班，去不了，下次。」

发完之后，他把手机揣回口袋，双手撑在围栏上，望着脚下的城市。

二十三层的高度，整个城市的夜景尽收眼底。远处的霓虹灯牌闪烁着，高架桥上车流如织，像血管里流动的光。风把他的头发吹得有些乱，他眯起眼睛，脑子里空空的。

这就是他的生活。

二十六岁，互联网公司内容审核员，月薪七千，租住在城中村的握手楼里。没有女朋友，没有存款，没有梦想。每天最大的追求就是准点下班和周末能睡到自然醒。

很无聊，但也很安全。

不需要承担什么，不需要对什么负责，不需要让任何人失望——因为从一开始就没有人对他抱有期待。

许昼安觉得这样挺好的。

屏幕亮了。

是一条短信，发送者的号码是一串零——000-0000-0000。

他摇了摇头，准备点击删除。2024年了，谁还信这种一看就是广告的玩意儿。他的手指移向屏幕右上角的删除键——

然后，他愣住了。

短信下方多出了一行字，不是预先显示的，而是此刻、在他眼前、一个字一个字地跳出来：

「倒计时：2分59秒。」

「2分58秒。」

「2分57秒。」

数字在跳动。

许昼安僵在原地。他不是被短信内容吓到了，而是——

这些字是怎么出现的？

他没有点开任何链接，没有下载任何程序，甚至连网都没有刷新。但那个倒计时就那么实时地、在他的屏幕上跳动着，每一秒都在减少。

这不可能是提前录制好的。

他低头看着自己的手。

手指僵硬地攥着手机，关节微微泛白，但无论他怎么用力，都没办法让手指松开。

这不对。

这绝对不对。

手机屏幕的光芒变得刺眼起来，那串数字仿佛从二维的屏幕里跃出，化作实体悬浮在他面前。冰蓝色的光芒，冷冰冰的，不带一丝温度。

不会吧？

「30秒。」

许昼安的心跳快到几乎要从嗓子眼里蹦出来。他拼命想让自己冷静——一定有什么合理的解释。也许是某种新型诈骗手段？全息投影技术？还是他加班加傻了产生幻觉？

但他的身体诚实地告诉他：这不是幻觉。

这是真的。

「9秒。」

「8秒。」

天台消失了。脚下的水泥地面化作虚无，他感觉自己正在坠落，但又没有任何坠落的风阻和失重感。四周变成一片黏稠的黑暗，只有那串冰蓝色的数字还在跳动。

「5秒。」

「4秒。」

「3秒。」

「2秒。」

「1秒。」

加班真的会死人的。

冰冷的机械女声在黑暗中响起。

许昼安猛地睁开眼睛。

他发现自己正躺在冰凉的大理石地面上，头顶是一个巨大的穹顶，四周是昏暗的灯光和一尊尊陈列在展柜里的雕塑。空气里弥漫着一股博物馆特有的气味——岁月、灰尘和防腐剂混合的味道。

熟悉。

太他妈熟悉了。

因为这里和他几个小时前审核的那张照片里的场景，一模一样。

他不是一个人。

恐慌的声音此起彼伏。

衔尾蛇。

无限的象征。

我就不该加班。

# 第二章 九个倒霉蛋和一座吃人的博物馆

「提示：目前存活玩家9人。天亮倒计时：6小时。」

冰冷的机械音播报完毕，全息投影缓缓消散，只留下昏暗的灯光和一群惊恐的人。

许昼安深吸一口气，强迫自己冷静下来。

恐慌没用。

无论眼前这一切是真是假——哪怕只有一丝可能是真的——恐慌都不会帮他活命。

他环顾四周，开始观察环境。

博物馆大厅是一个圆形空间，穹顶很高，目测至少有三层楼。四周是八个通向不同方向的拱形门洞，每个门洞上方都有一块石质匾额，刻着不同的名字——"东方厅"、"西方厅"、"古代厅"、"现代厅"之类。

但问题是：所有的门洞里都是一片漆黑，看不到尽头，像是八张黑洞洞的大嘴。

而且——

许昼安皱起眉头。

他刚才醒来的时候，隐约看到大厅墙壁上有门的轮廓。但现在仔细看……那些门不见了。

墙壁是一整块完整的大理石，连缝隙都没有。

出口消失了。

"冷静一下，大家都冷静一下！"

一个声音打断了他的思绪。许昼安转头看去，是一个戴眼镜的年轻男人，看起来二十七八岁，穿着休闲西装，努力让自己显得镇定。

"我叫李正阳，我是心理咨询师，大家先别慌……"

"别慌？！"睡衣大妈的声音尖锐得能划破玻璃，"让我死我怎么不慌？！这到底是哪儿？谁把我弄来的？我要报警！我要……"

"手机没信号。"许昼安平静地说。

大妈愣住，赶紧去翻自己的手机——果然，屏幕上只有雪花噪音和那个诡异的衔尾蛇图标。

"我操！什么破玩意儿！"黄毛小伙把手机往地上一摔，"谁他妈绑架老子！有种出来！"

没有人出来。

博物馆安静得像一座坟墓。

许昼安没有理会那些陷入恐慌的人，而是悄悄往大厅边缘移动，靠近一尊展柜里的雕塑。

这是一尊青铜人像，造型古朴，闭着眼睛，双手合十，看起来像是某个朝代的祭祀用品。展柜玻璃很厚，灯光从下方打上来，把人像的影子投在天花板上。

但许昼安注意到的不是人像本身，而是展柜旁边的介绍牌。

「祈祷者·青铜」

「朝代：不明」

「出土地点：不明」

「特性：据说在午夜时分会睁开眼睛，注视最近的活人。」

……

许昼安盯着"特性"那一行看了三秒，然后默默后退了两步。

介绍牌在提示他。

"展品会苏醒"——那个机械音说的不是废话，而是字面意思。

他快速扫视周围的其他展柜。油画、瓷器、木雕、石像……每个展品旁边都有一块介绍牌，而每块介绍牌的"特性"栏都写着类似的内容：

「据说会在有人靠近时转动脖子。」

「据说会吞噬直视它超过十秒的人。」

「据说会模仿观察者的动作，但总是慢半拍。」

这些介绍牌是线索，也是规则。

许昼安脑子转得飞快。既然这是一个"游戏"，那就一定有通关的方法。机械音说要找到"真正的出口"——这说明存在一个出口，只是被隐藏了。

而线索……

很可能就藏在这些介绍牌里。

"喂，你在看什么？"

一个冷淡的声音从背后传来。许昼安转头，看到一个年轻女人正站在不远处，目光锐利地盯着他。

她大概二十三四岁，黑色长发扎成马尾，穿着一件简单的白T恤和牛仔裤，长相算不上惊艳但五官很精致。最引人注意的是她的眼神——冷静、分析、带着一种让人不舒服的审视感，像是在看一个实验对象。

"介绍牌。"许昼安如实回答。

"你发现了什么？"

"还不确定。"许昼安顿了顿，"你是谁？"

"沈听澜。"女人简短地报了名字，目光越过他，落在那块介绍牌上，"神经生物学博士在读。你呢？"

"许昼安。内容审核员。"

"……什么？"

"就是审核互联网上那些乱七八糟内容的。"许昼安耸耸肩，"没你那个头衔好听。"

沈听澜沉默了一秒，似乎在评估他是不是在开玩笑。最终她没有继续这个话题，而是蹲下身，开始研究展柜旁边的介绍牌。

"这些特性描述不像是随便写的。"她喃喃道，"'据说'是一种不确定性表述，但如果这里真的像那个声音说的那样、展品会'苏醒'的话……这些特性可能就是实际的攻击模式。"

许昼安挑了挑眉。

这女的脑子转得挺快。

"所以你觉得这些介绍牌是攻略？"他问。

"不完全是攻略，更像是……"沈听澜站起身，目光在大厅里扫了一圈，"规则说明。这个地方有自己的规则，介绍牌在告诉我们这些规则是什么。但——"

她皱起眉头。

"'真正的出口'是什么意思？"

许昼安也在想这个问题。

既然有"真正的出口"，那就意味着存在"假的出口"——或者说，看起来像出口但实际上不是的东西。

他看向那八个黑洞洞的门洞。

"你觉得那些门通向哪里？"

"不知道。"沈听澜坦诚地说，"但我不建议贸然进去。在没有搞清楚规则之前，任何行动都可能致命。"

两人对视一眼，达成了某种默契。

就在这时，大厅另一边传来一阵骚动。

"我不管了！我要出去！"

是那个睡衣大妈。她不知道从哪里鼓起勇气，正跌跌撞撞地往其中一个门洞跑去。几个人想拦她，但她根本不听，尖叫着冲进了黑暗里。

"等等！别进去——"心理咨询师李正阳喊道。

太晚了。

大妈的身影消失在门洞的黑暗中。

大厅里一片死寂。

所有人都屏住呼吸，等待着什么。

一秒。

两秒。

三秒。

然后，黑暗深处传来一声撕心裂肺的惨叫。

那声音持续了大约两秒，戛然而止。

接着，机械女声再次响起：

「提示：存活玩家8人。死亡原因：进入未开放区域。」

许昼安感觉胃里一阵翻涌。

他转头看向沈听澜，发现她的脸色也白了几分——但她的眼神依然冷静，甚至带着一丝……观察的意味。

"门洞是陷阱。"她低声说，"至少在满足某个条件之前是。"

"某个条件？"

"'找到真正的出口'。"沈听澜念出了那句规则，"这说明出口不是一开始就存在的，而是需要被'找到'。也就是说……"

"出口可能藏在展品里。"许昼安接话。

两人再次对视。

这一次，他们的眼神里都多了一点东西——不是信任，而是某种基于利益的默契。

"我们需要分头行动。"沈听澜说，"把所有展品的介绍牌都看一遍，找出规律。"

# 第三章 老油条的生存法则

青铜人像的眼睛睁开了。

那是一双没有瞳孔的眼睛，像两颗空洞的玻璃珠子，正缓缓转动，似乎在寻找什么。

许昼安屏住呼吸，一动不动。

"这尊人像的特性是'注视最近的活人'。"她用几乎听不见的声音说，"也就是说，它只会注视，不会攻击。"

"不确定。但介绍牌上只写了'注视'，没写别的。"

许昼安咬了咬牙。

赌一把。

他缓缓后退一步，然后又退一步。青铜人像的头跟着他的动作微微转动，那双空洞的眼睛始终"看"着他——但除此之外，它没有任何其他动作。

两米。

三米。

五米。

许昼安退到了一个相对安全的距离。人像的头转到了极限角度，然后停了下来，像是一个没接到命令的玩偶。

"成了。"他松了一口气。

但下一秒，大厅里响起了一声惨叫。

黄毛只来得及喊出这两个字，整个人就被拖进了展柜里。

玻璃合上的瞬间，展柜内部亮起一道刺眼的白光。等光芒散去，许昼安看到展柜里多了一尊新的雕塑——姿势扭曲、表情痛苦、头发染成黄色的石像。

「提示：存活玩家7人。死亡原因：触发展品特性。」

机械女声冷漠地播报。

许昼安感觉胃在抽搐。

他刚才审核那张照片时看到的"玻璃展柜里的人脸"，原来就是这么来的。

那些展品……是人变的。

恐慌像瘟疫一样蔓延。剩下的五个人（除了他和沈听澜）已经完全丧失了理智，四处乱窜，有的在哭喊，有的在求救，有的在往门洞里跑——

"不要进门洞——"

太晚了。

又一个人消失在黑暗里，然后是熟悉的惨叫声，然后是机械女声的死亡播报。

「提示：存活玩家6人。」

六个人了。

"跟我来。"

一个四十来岁的中年男人，就是之前靠在墙角、一言不发的那个。

这人知道展品的特性。

老玩家。

他怂到不敢乱跑，只敢躲在原地观察。

结果反而活了下来。

这算什么？苟道真的能续命？

"除非你通关一百个副本。"宋时行顿了顿，"传说通关一百个副本就能获得自由，脱离这个系统。但……"

"但什么？"

"我通关了四十七个，没见过任何人真的做到过。"宋时行的语气很平淡，像是在陈述一个无关紧要的事实，"那些说自己要通关一百个副本获得自由的人，最后都死了。"

许昼安感觉心里像是被什么东西压住了。

四十七个副本。

他刚刚通关了第一个，差点就死在里面，而这个男人已经通关了四十七个。

四十七次和死亡擦肩而过。

"积分有什么用？"沈听澜的声音打断了他的思绪。

她的语气依然冷静，像是在讨论一个学术问题而不是生死攸关的事情。许昼安不知道该佩服她的心理素质还是该怀疑她是不是神经有问题。

"积分可以兑换道具。"宋时行从口袋里掏出手机——上面已经不再是雪花噪音了，而是一个全新的界面，以那条衔尾蛇为logo。"这是玩家APP，里面有商城系统。急救包、解毒剂、临时护盾、情报文件……什么都有，只要你出得起积分。"

他把手机递给许昼安。

许昼安接过来，低头看着屏幕。

商城界面很简洁，分成几个大类：「生存道具」「情报资料」「特殊物品」「服务」。他随手点开"生存道具"一栏，看到第一个商品——

「初级急救包」

「效果：快速止血，恢复少量生命值」

「价格：50积分」

他又往下滑了滑。

「中级急救包」——100积分。

「高级急救包」——300积分。

「复活道具·一次性」——10000积分。

"……"

许昼安沉默地关掉商城，把手机还给宋时行。

他的200积分，在这个商城里屁都不是。

"现在你明白了吧？"宋时行把手机揣回口袋，"这个系统的设计目的就是让你不断参与副本、不断消耗、不断挣扎。积分是唯一的货币，而货币永远不够用。"

"那你还活着是怎么回事？"许昼安问，"四十七个副本，总不可能每次都白嫖吧？"

"我运气好。"宋时行淡淡说，"而且我很早就发现了——在副本里，脑子比道具更重要。"

许昼安想起了刚才的评分规则，若有所思。

"对规则的理解程度"——这是决定评分的关键。

换句话说，这个系统奖励的不是战斗力、不是勇气、不是运气，而是……分析能力。

越会钻研规则的人，评分越高。

越会利用规则的人，活得越久。

这倒是很符合他的风格。

他这辈子最擅长的事情就是摸鱼——而摸鱼的精髓是什么？是找到系统的漏洞，用最小的付出获得最大的收益。

也许……他真的能在这里活下去？

"你们在想什么？"宋时行突然说，"休息时间快结束了，马上就要被送回现实世界。我最后说几件事——"

他看着许昼安和沈听澜，表情严肃起来。

"第一，不要对任何人透露无限流的存在。说了也没人信，只会让你看起来像精神病。"

"第二，下次副本的倒计时是七天。这七天里，你们可以正常生活，但不要试图逃跑。系统会追踪你的位置，逃跑等于送死。"

"第三——"他顿了顿，"记住我之前说的条件。出了这里，我会联系你们。"

话音刚落，白色空间开始扭曲。

许昼安感觉眼前一花，然后——

熟悉的夜风灌进来。

他站在公司天台上，手里还攥着那部手机。

四周一切如常。远处的城市灯火闪烁，脚下是熟悉的水泥地面，空气里有初秋的凉意。

他低头看手机——

时间显示：23:48。

就在他收到那条短信之后……一秒钟。

只有一秒钟。

他在那个博物馆里待了好几个小时、看着四个人死去、和两个陌生人组队逃亡、最后跳进一幅油画里……

现实世界只过了一秒钟。

许昼安抬头看着夜空，忽然觉得有些恍惚。

刚才发生的一切是真的吗？

会不会只是他加班太累产生的幻觉？

他低头，打开手机。

主屏幕上，那个衔尾蛇的APP图标安静地躺在那里，存在感强得刺眼。

他点开APP。

界面跳出来——

「玩家：许昼安」

「等级：S」

「积分：200」

「通关副本：1」

「下次副本倒计时：6天23小时59分48秒」

是真的。

这一切都是真的。

他被拖进了一个叫做"无限流"的死亡游戏，每隔七天就要参加一次可能致命的副本，不能逃跑、不能退出，唯一的结局是要么死在里面、要么通关一百个副本。

许昼安盯着那个倒计时，忽然笑了一声。

笑声很轻，带着一丝自嘲。

"挺好的。"他喃喃道，"至少不用担心加班了。"

……

回到出租屋的时候已经是凌晨两点。

许昼安没有立刻睡觉，而是躺在床上，盯着天花板发呆。

他在想那行字。

「许昼安，欢迎回来。」

刻在画框上的那行字，像一根刺一样扎在他脑子里，怎么都拔不出来。

为什么是"回来"？

他从来没去过那里。他第一次参加无限流的副本，第一次进入那座博物馆，第一次看到那幅油画——

但那行字的意思很明确：他不是"来"，是"回来"。

有人在等他。

有人早就知道他会出现在那里。

这种感觉让他很不舒服。

许昼安翻了个身，从枕头底下摸出手机，打开那个衔尾蛇APP，开始仔细研究里面的功能。

商城他已经看过了。

除了商城，APP里还有几个板块——「个人信息」「副本记录」「排行榜」「社交」「档案馆」。

他一个一个点开看。

「个人信息」里是他的基础数据，没什么特别的。

「副本记录」里显示他只通关了一个副本「午夜博物馆·入门试炼」，评级S。

「排行榜」里显示的是当前活跃玩家的积分排名。他扫了一眼榜首——一个叫"陈焕"的玩家，积分12700。

一万多积分……这要通关多少副本才能攒到？

「社交」板块允许玩家互相添加好友和私聊。他发现自己的好友列表是空的，但有两条待处理的好友申请——来自"沈听澜"和"宋时行"。

他想了想，都通过了。

最后是「档案馆」。

点开之后，界面上弹出一条提示：

「档案馆功能需要消耗积分使用。每次查询消耗50积分。是否继续？」

许昼安犹豫了一下。

他现在总共就200积分，查询一次就要消耗四分之一。但……

他想起了那行字。

「许昼安，欢迎回来。」

如果答案在档案馆里，他必须得查。

他点了「确认」。

界面刷新，变成一个类似搜索引擎的页面。搜索框上方写着：「请输入您要查询的内容（支持关键词/姓名/副本名称）」

许昼安在搜索框里打了三个字：

「许昼安」

搜索结果加载了几秒钟，然后——

「未找到相关结果。」

什么？

许昼安愣住了。

他以为至少能搜到自己刚刚的通关记录，但系统显示"未找到相关结果"。

难道搜索不包括自己？

他换了个关键词：

「午夜博物馆」

这次有结果了。

屏幕上显示出一长串记录，都是不同时间、不同玩家通关「午夜博物馆」副本的数据。他随手点开几条看了看，都是些干巴巴的统计数字——参与人数、存活人数、通关时间、最高评分之类。

没什么有用的。

他又换了个关键词：

「欢迎回来」

搜索中……

「未找到相关结果。」

还是没有。

许昼安皱起眉头。

他输入了最后一个关键词——

「零号副本」

这个词是他临时想到的。之前宋时行提到过"入门试炼"这个字眼，那就意味着还有更高级的副本。如果存在"一号副本"、"二号副本"，那会不会有"零号副本"？

搜索结果加载……

加载……

加载……

然后，屏幕上跳出一条记录。

只有一条。

「零号副本·创世试炼」

「状态：已封存」

「最后一次开启时间：15年前」

「通关者名单：（点击展开）」

许昼安的手指悬停在"点击展开"上方，犹豫了一秒钟，然后点了下去。

名单加载出来。

一共五个名字，按照顺序排列：

「1. 林幺幺（状态：特殊）」

「2. 方时序（状态：未知）」

「3. 许鹤铭（状态：未知）」

「4.」

许昼安的目光定在第三个名字上，整个人僵住了。

许鹤铭。

那是他父亲的名字。

他的父亲——那个在他十一岁时突然失踪、再也没有回来、被警方定性为"失踪人口"的男人——出现在一份十五年前的"副本通关者名单"里。

这怎么可能？

他父亲和这个无限流游戏有什么关系？

他父亲的失踪……和这一切有关吗？

许昼安盯着那三个字看了很久，脑子里一片混乱。

他想起了画框上的那行字——

「许昼安，欢迎回来。」

现在他明白了。

那句话不是对他说的。

是对他父亲说的。

而他……继承了父亲的位置。

手机"叮"地响了一声，一条私信弹出来：

「沈听澜：你有没有在档案馆查到什么？」

许昼安盯着这条消息，手指悬在屏幕上方，久久没有动作。

他该告诉她吗？

告诉她他父亲的名字出现在十五年前的副本通关名单里？

告诉她他从出生起就和这个该死的游戏有关联？

他不知道。

他什么都不知道。

窗外，天快亮了。城市的轮廓在晨曦中渐渐清晰，新的一天即将开始。

但对许昼安来说，一切都不一样了。

他不再是一个普通的摸鱼打工人。

他是一个玩家。

而且，似乎是一个从出生就被选中的玩家。

许昼安躺在床上，盯着天花板，喃喃自语：

"爸，你到底卷进了什么破事儿……"

没有人回答他。

只有那个APP的倒计时在默默跳动——

6天22小时47分30秒。
