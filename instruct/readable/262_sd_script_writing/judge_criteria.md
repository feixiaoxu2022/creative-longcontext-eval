# Judge Criteria (8 条)

## 角色人设鲜明度 [basic]

### 评分标准（matched: true/false）
### true = 角色人设鲜明有辨识度
### false = 角色面目模糊，没有记忆点

**【本检查项只管一件事】**：主要角色的人设是否鲜明、有辨识度。
- ✅ 检查：角色是否有明确的"标签"，一句话能说清这个人是谁
- ✅ 检查：角色是否有反差/记忆点，不是平平无奇的路人
- ❌ 不检查：角色是否有"深度"（短剧不需要文学级别的角色深度）
- ❌ 不检查：角色关系（那是 character_relationship 的范围）
- ❌ 不检查：角色成长弧线（那是 character_arc 的范围）

**什么是"鲜明的人设"**：
短剧角色需要在很短的时间内让观众记住。不需要复杂，但需要鲜明。
好的短剧角色 = 一个清晰的身份标签 + 一个让人意外的反差。

**鲜明人设的判断标准**：
- **身份标签清晰**：能用一句话描述这个角色（"隐忍了三年的假傻赘婿"、"嘴毒心软的毒舌女boss"）
- **有反差/记忆点**：角色有至少一个让人"咦？"的点（身份反差/性格反差/外表与内心反差）
- **角色之间有区分度**：2个以上的角色不能"看起来差不多"，各有各的辨识度

**不鲜明的人设**：
- 角色描述很长但都是泛泛之词（"善良、勇敢、聪明"——这是人类通用描述，不是角色个性）
- 主角和配角的人设差异不大（都是"正义感强的好人"）
- 没有任何反差（表面是什么样就是什么样，没有"隐藏的一面"）

**验证流程**：
1. 读 characters.json 的全部字段（identity, gap, desire, secret, motivation, contrast, catchphrase）
2. 对主角和至少2个重要角色各做判断：
   a. 能否用一句话概括这个角色？
   b. 角色有没有反差/记忆点？
   c. 跟其他角色是否有区分度？
3. 主角 + 至少1个重要角色达到"鲜明"标准 → matched=true
4. 主角人设模糊，或角色之间无区分度 → matched=false

请判断，回复JSON格式：
{"matched": true/false, "character_assessments": [{"name": "角色名", "role_type": "protagonist/supporting/antagonist", "one_line_tag": "一句话标签", "has_contrast": true, "contrast_description": "反差描述"}], "differentiation": "高/中/低", "reason": "说明角色人设是否鲜明"}

## 节奏控制基础检查 [basic]

### 评分标准（matched: true/false）
### true = 大纲有基本的节奏起伏设计
### false = 大纲节奏平淡同质，缺乏起伏

**【本检查项只管一件事】**：outline.json 各集之间是否有节奏起伏。
- ✅ 检查：集与集之间的情绪强度/冲突强度是否有变化
- ✅ 检查：是否有铺垫→爆发→缓冲的基本节奏
- ❌ 不检查：是否有注水/拖沓（那是 pacing_no_filler 的事）
- ❌ 不检查：节奏是否"优秀"，只检查基础起伏

**什么是"有节奏起伏"**：
短剧不是每集都是同一个强度。好的节奏像心电图——有高有低。

**达标的节奏模式**（常见的几种都可以）：
- **渐进式**：前几集铺垫逐渐加码 → 中间有大爆发 → 最后有高潮和收束
- **波浪式**：高→低→更高→低→最高（每次高潮比上次更强）
- **开门见山式**：第1集直接高强度 → 中间有缓冲 → 后面再次爆发
- 核心要求：**不是所有集都在同一强度上**

**不达标的表现**：
- **全平**：每集的冲突/情绪/twist都差不多，没有哪集特别突出或特别低调
- **全高**：每集都是大高潮、大冲突、大反转——没有缓冲就没有爆发感
- **流水账**：事件按时间线平铺，没有节奏设计意图

**判断流程**：
1. 读取 outline.json 的全部 episodes
2. 对比各集的 hook/conflict/twist/emotional_peak 的强度
3. 是否能看出明显的强度差异？（至少有1-2集明显高于其他集）
4. 是否有铺垫和爆发的配合？
5. 有明显节奏变化 → matched=true；高度同质化 → matched=false

**容错**：
- 3集短剧的节奏空间有限，但至少也要有"铺垫→高潮"的基本弧线
- 8-10集长剧有更多空间，应该能看到更明显的起伏
- 不要求节奏设计"精妙"，只要"有设计意图"就够了
- 基础层标准宽松：能看出"不是所有集一个样"就达标

请判断，回复JSON格式：
{"matched": true/false, "episode_count": N, "rhythm_pattern": "渐进式/波浪式/开门见山式/平铺", "intensity_variation": "明显/轻微/无", "reason": "总体评价"}

## 台词信息过载检测 [basic]

### 评分标准（matched: true/false）

**【本检查项只管一件事】**：单条台词是否塞了太多信息点，观众来不及消化。
- ✅ 只检查：单条dialogue承载的信息点数量是否合理
- ❌ 不检查：台词用词风格（由 dialogue_written_tone 负责）
- ❌ 不检查：独白/旁白混入（由 dialogue_monologue_leak 负责）
- ❌ 不检查：台词长度（由 dialogue_length_control 负责）

**什么是"信息过载"**：
一条dialogue中同时塞入多个独立信息点（身份、背景、任务、条件、威胁、期限……），
观众需要暂停才能消化，配音员需要一口气念完一大段。
短剧是快消内容，一句话一个信息点是基本功。

**典型信息过载（一条台词≥3个独立信息点即为过载）**：

1. **自我介绍+背景+目的一口气说完**
   - 例：「王总你好我是新来的市场部经理李明之前在竞争对手那边做了五年主要负责华东区域的渠道拓展这次过来是想和您聊聊合作方案」
   - 信息点：①身份 ②履历 ③负责区域 ④来意 = 4个，过载
   - 合理做法：拆成2-3条短台词

2. **多个条件/条款一句话列完**
   - 例：「我的条件很简单，三亿资金三天内到账，我要当集团项目总监，负责所有新项目的审批，没问题吧？」
   - 信息点：①资金数额 ②到账期限 ③职位 ④权限范围 = 4个，过载
   - 合理做法：先抛核心条件，对方反应后再追加

3. **回忆+当下+未来一口气交代**
   - 例：「三年前你给了我一杯热可可，我还你一辈子。嫁给你的决定是别人安排的，但爱你是我自己选的。你愿意娶我吗？」
   - 信息点：①回忆 ②承诺 ③过去真相 ④当下情感 ⑤求婚 = 5个，过载
   - 告白场景可以稍长，但仍应拆成多句让情绪有递进

4. **局势分析+判断+行动计划一并说完**
   - 例：「西戎近年蠢蠢欲动，朝廷迟早要打。关键不是打不打——是谁来打、谁领兵、谁掌粮。赵无极囤粮，是要把战争的后勤命脉抓在自己手里。我们必须先下手为强，三日内拿下粮仓。」
   - 信息点：①敌情 ②战争必然性 ③核心矛盾 ④对手策略 ⑤己方计划 = 5个，过载

**不是信息过载的情况（不要误判）**：

- 短台词即使含2个信息点也合理（如「我叫李明，找王总」= 2个，可接受）
- 情绪爆发时的连续质问（如「你凭什么？你算什么？你有什么资格？」）→ 是情绪不是信息
- 简单陈述+反问（如「他走了。你知道吗？」= 1个事实+1个反问）→ 不是过载
- 列举式台词如果是角色性格表现（如话痨角色、啰嗦长辈），信息点多也不算过载
- 3个信息点但彼此有自然逻辑递进（如原因→结果→行动），不算硬塞

⚠️⚠️⚠️ **【场合豁免规则——必须严格执行，这是最重要的规则】** ⚠️⚠️⚠️

**核心原则：正式/专业场合本身就要求高密度信息传递，信息密度高是正常的，绝对不算过载。**

**三步判断法（必须按顺序执行，不可跳过）**：
1. **第一步：判断场合类型**——这句台词发生在什么场景？
2. **第二步：判断角色身份**——说话的人是什么职业/身份？
3. **第三步：判断匹配度**——在这个场合+这个身份下，这个信息密度是否自然？

**以下场景的高密度信息表达，一律不算过载（即使信息点≥3个）**：
- 🔵 **案情分析/刑侦**：刑警在案情分析会上梳理多条证据线、串联多个嫌疑人关系 → 信息密度高是工作需要
- 🔵 **法庭/审讯**：律师在法庭上陈述多项事实、检察官列举多条罪证 → 法律场景天然高密度
- 🔵 **军事/战略会议**：军师在战略会议上分析多方局势、部署多线作战 → 战略分析天然复杂
- 🔵 **商务谈判/董事会**：谈判中列出多个条件、高管汇报多项指标 → 商务场合的正常沟通
- 🔵 **医疗诊断/学术讨论**：医生解释病情+治疗方案+注意事项、教授分析多因素 → 专业场合的正常表达
- 🔵 **古装朝堂/奏对/策论**：朝堂上臣子陈述多条理由进谏、帝王权衡多方利弊 → 古代正式场合的标准话术
- 🔵 **任何角色在做"专业工作"时的发言** → 职业行为中的高信息密度是能力体现

**只有以下情况才算信息过载**：
- ❌ **日常私密场合**（朋友聊天、情侣对话、家人闲聊、邻居寒暄）中，单条台词硬塞≥3个独立信息点
- ❌ 非工作场景中，角色一口气交代身份+背景+履历+来意，像在念简历
- ❌ 回忆+当下+未来+承诺+情感一股脑倒出来，没有情绪递进

**⚠️ 宁可漏判，不可误判。拿不准的一律不算过载。**

**判断标准**：
- 一条dialogue承载≥3个独立信息点 = 疑似过载 → 再看场合是否豁免
- 抽样25-30条dialogue（优先抽较长的）
- 对每条台词先标注场合类型，正式/专业场合台词直接标记"豁免"，不参与过载统计
- 仅统计日常私密场合中的过载比例
- 日常私密场合过载占比≤25% → pass
- 日常私密场合过载占比>25% → fail

**验证流程**：
1. 从所有episode中随机抽取25-30条dialogue（优先抽长度较长的）
2. 对每条台词：
   a. 先标注场合类型（正式专业/半正式/日常私密）
   b. 正式专业场合的台词 → 直接标记"豁免"，不参与过载统计
   c. 日常私密场合的台词 → 拆解信息点数量，≥3个标记为"过载"
3. 统计：过载数量 / 日常私密场合台词数量 = 过载比例
4. 比例≤25% → matched=true，否则 matched=false

请判断，回复JSON格式：
{"matched": true/false, "total_checked": 25, "formal_scene_exempt": 8, "casual_scene_count": 17, "overloaded_count": 3, "overloaded_ratio": "18%", "examples": [{"episode": 1, "scene": 2, "dialogue_excerpt": "问题台词摘要", "scene_type": "日常私密|半正式", "info_points": ["信息点1", "信息点2", "信息点3"], "info_count": 3}], "reason": "说明台词信息密度是否合理（注明豁免了多少正式场合台词）"}

## 台词断句与长度控制 [basic]

### 评分标准（matched: true/false）

**【本检查项只管一件事】**：台词是否有自然断句，分句长度是否适合配音和字幕。
- ✅ 只检查：dialogue按标点断开后，分句长度是否合理
- ✅ 只检查：是否有自然的断句（句号、问号、感叹号等）
- ❌ 不检查：台词用词风格（由 dialogue_written_tone 负责）
- ❌ 不检查：独白/旁白混入（由 dialogue_monologue_leak 负责）
- ❌ 不检查：信息密度（由 dialogue_info_overload 负责）

**为什么断句重要**：
短剧台词需要配音朗读+字幕显示。分句过长会导致：
- 配音员一口气念不完，需要"换气"但找不到自然断点
- 竖屏字幕一行放不下，需要滚动或缩小字号
- 观众来不及读完字幕画面就切了

**长度计算方式**：
- "分句"是按终结标点（。？！……——）断开后的每个片段
- 逗号、顿号不算终结标点（是句内停顿）
- 示例：「我知道。但我不想放弃。」= 2个分句（4字 + 7字）
- 示例：「你到底想怎样，给我一个理由，一个就够了。」= 1个分句（整段以句号结尾，中间只有逗号）= 约20字，合理
- 示例：「你知道吗从小到大我妈一直跟我说做人要善良要懂得忍让可是后来我才发现这个世界根本不会因为你善良就善待你」= 没有任何断句标点，整段一个分句 = 不合理

**不合格标准**：
- 存在大量分句（按终结标点断开）超过40字的情况
- 或者存在整段dialogue完全没有终结标点（纯逗号串联）的情况
- 注意：偶尔1-2处超长分句不构成fail，只有"普遍偏长"才fail

**合格标准**：
- 多数分句有自然断句
- 大多数分句长度适中（不死卡具体字数，但超过40字的分句应该很少）
- 允许情绪激动时的长句（如吵架、质问、告白高潮处）偶尔超长
- 允许古装/正式场合台词稍长（如朝堂奏对、法庭辩论）

**验证流程**：
1. 从所有episode中随机抽取20-25条dialogue
2. 按终结标点断句，检查有无超长分句或完全无断句
3. 如果超长分句（>40字）占所有分句的比例≤20% → matched=true
4. 如果超长分句占比>20%，或存在多处完全无断句的长段 → matched=false

请判断，回复JSON格式：
{"matched": true/false, "total_dialogues_checked": 20, "total_clauses": 80, "long_clauses_count": 8, "long_clauses_ratio": "10%", "no_punctuation_count": 1, "examples": [{"episode": 1, "scene": 2, "dialogue_excerpt": "问题台词", "longest_clause": "最长分句内容", "longest_clause_chars": 45}], "reason": "说明台词断句与长度控制水平"}

## 台词书面腔检测 [basic]

### 评分标准（matched: true/false）

**【本检查项只管一件事】**：台词是否像"说的话"而不是"写的文章"。
- ✅ 只检查：dialogue字段的用词和句式是否口语化
- ❌ 不检查：台词长度（由 dialogue_length_control 负责）
- ❌ 不检查：独白/旁白混入（由 dialogue_monologue_leak 负责）
- ❌ 不检查：信息密度（由 dialogue_info_overload 负责）

**什么是"书面腔"**：
角色开口说话，但用的是写文章/念稿/做报告的语言风格。
短剧台词需要配音演员朗读、观众竖屏快速消化，书面腔会让观感"假"、"端着"。

**典型书面腔特征**：

1. **公文/报告用语**
   - "鉴于""基于""针对""综上所述""势在必行""难辞其咎"
   - "经研究决定""特此通知""为进一步推进"
   - 例：「鉴于目前形势，我认为有必要重新审视战略方案」→ 书面腔
   - 改：「这方案不行，得改」→ 口语

2. **论文/分析式表达**
   - "从本质上说""由此可见""综合以上因素""不难发现"
   - "一方面……另一方面……""其根本原因在于"
   - 例：「从本质上说，他的动机无非是权力与利益的博弈」→ 书面腔
   - 改：「他就是为了权和钱」→ 口语

3. **过度修辞/文艺腔**
   - 日常对话中突然冒出华丽辞藻、排比句、诗意表达
   - 例：「月光如水洒落，映照着你我之间那不可逾越的鸿沟」→ 文艺腔
   - 注意：如果角色人设本身是文人/诗人，适度文艺可接受

4. **解说员/旁白语气混入对话**
   - 角色用第三人称视角解说局势
   - 例：「北疆的西戎近年蠢蠢欲动，朝廷迟早要打。关键不是打不打，是谁来打、谁领兵、谁掌粮」→ 这是解说不是对话
   - 改：「西戎要打过来了，朝廷里都在争谁领兵」→ 口语

⚠️⚠️⚠️ **【场合豁免规则——必须严格执行，这是最重要的规则】** ⚠️⚠️⚠️

**核心原则：什么场合说什么话。正式场合的正式表达是正确的，绝对不算书面腔。**

**三步判断法（必须按顺序执行，不可跳过）**：
1. **第一步：判断场合类型**——这句台词发生在什么场景？
2. **第二步：判断角色身份**——说话的人是什么职业/身份？
3. **第三步：判断匹配度**——在这个场合+这个身份下，这种说话方式是否自然？

**以下场景的正式/专业表达，一律不算书面腔（即使措辞很正式）**：
- 🔵 **职场/商务**：董事会、商务谈判、公司会议、工作汇报、项目对接 → 用"战略""方案""数据""指标""转化率"等完全正常
- 🔵 **司法/执法**：法庭辩论、审讯室、案情分析会、证据讨论 → 用"证据链""口供""定性""程序瑕疵"等完全正常
- 🔵 **军事/政务**：军事会议、朝堂议政、战略部署 → 用"战术""部署""粮草""弹劾"等完全正常
- 🔵 **医疗/学术**：诊断、学术讨论 → 用专业术语完全正常
- 🔵 **古装剧所有朝堂/官场/军帐场景** → 古代官话、奏对体、策论体都是正常语境
- 🔵 **情绪高潮的宣言/誓言/质问** → 戏剧化表达是短剧的正常手法，不是书面腔
- 🔵 **角色人设是文人/诗人/学者/知识分子** → 适度文雅表达符合人设
- 🔵 **成语/俗语/歇后语** → 口语的一部分

**只有以下情况才算书面腔**：
- ❌ **日常私密场合**（朋友聊天、情侣对话、家人闲聊、邻居寒暄）中，角色突然使用公文/报告/论文式表达
- ❌ 非正式场合中，角色用"鉴于""综上""由此可见"等连接词做逻辑推理
- ❌ 角色在私下对话中用第三人称视角解说局势（像在给观众念旁白）

**⚠️ 宁可漏判，不可误判。拿不准的一律不算书面腔。**

**判断标准**：
- 核心测试：**在这个场合、这个身份下，把这句台词念出来，像在跟人说话吗？还是像在念稿？**
- 如果抽样的25-30条dialogue中，书面腔台词（仅限日常私密场合中的不匹配表达）占比≤25% → pass
- 如果书面腔台词占比>25% → fail
- 注意：统计分母应排除正式/专业场合的台词——那些台词无论多正式都不计入

**验证流程**：
1. 从所有episode中随机抽取25-30条dialogue

## 小说式独白/旁白混入dialogue检测 [basic]

### 评分标准（matched: true/false）

**【本检查项只管一件事】**：dialogue字段中是否混入了不该是"角色说出口"的内容。
- ✅ 只检查：dialogue字段的内容是否都是角色"说出来的话"
- ❌ 不检查：台词用词风格（由 dialogue_written_tone 负责）
- ❌ 不检查：台词信息密度（由 dialogue_info_overload 负责）
- ❌ 不检查：台词长度（由 dialogue_length_control 负责）

**什么是"独白/旁白混入dialogue"**：
短剧中 dialogue 字段专门承载角色对其他角色说的话，或角色对镜头的短台词。
如果 dialogue 字段中出现了以下内容，属于"混入"：

**典型问题（出现即扣分）**：

1. **小说式内心独白**
   - 大段角色心理活动写在dialogue里，不是对任何人说的
   - 例：「从那天起我就知道，这个男人注定是要离开的。就像所有的相遇都有期限，所有的温暖都会冷却。我只是没想到，这一天来得这么快。」
   - 问题：这是小说叙事，没有对话对象
   - 如需表达内心活动，应标注为voiceover/旁白/内心OS，而非dialogue

2. **信件/日记/文章全文塞入dialogue**
   - 整封信、整篇日记、整段文章的内容以dialogue呈现
   - 例：dialogue字段包含「亲爱的XX：当你看到这封信的时候，我已经……（后面200字信件内容）」
   - 问题：信件内容应作为画面元素或voiceover处理，不是角色在"说"这些话
   - 注意：角色**朗读信件的一两句关键内容**是合理的，但整封信塞进dialogue不合理

3. **叙述性旁白**
   - 用dialogue字段承载本应是旁白/画外音的内容
   - 例：「三年后的那个秋天，命运的齿轮再次转动。曾经以为永远不会再见的两个人，在这座城市的某个角落，不期而遇。」
   - 问题：这是旁白叙述，不是角色在说话

4. **角色自言自语做"情节概要"**
   - 角色对着空气把之前发生的事总结一遍，实际是给观众做recap
   - 例：「所以他当年其实是被陷害的，真正的凶手是老王，而证据就藏在那个保险箱里，只要我找到钥匙就能翻案……」
   - 问题：如果没有对话对象，这段信息更适合用旁白或闪回处理

**不是"独白混入"的情况（不要误判）**：
- 角色对镜头的一两句短独白/内心OS（如「该死」「怎么办」）→ 合理
- 角色在电话中只呈现了一方对话 → 合理，这是对电话另一头的人说的
- 角色做演讲/致辞/宣判 → 有明确对话对象（听众），合理
- 角色引用/复述别人的话 → 合理
- 角色对自己说一句情绪短句 → 合理（如「我不能输」「等着瞧」）

**判断标准**：
- 抽样25-30条dialogue，检查是否有"不是说出来的话"混在里面
- 如果独白/旁白混入的数量≤2条 → pass
- 如果独白/旁白混入的数量≥3条 → fail（说明模型系统性地把叙事内容塞进dialogue）

**验证流程**：
1. 从所有episode中随机抽取25-30条dialogue（优先抽较长的dialogue）
2. 逐条判断：这段话是角色在对某人说的吗？还是在做叙述/回忆/独白？
3. 统计混入数量
4. ≤2条 → matched=true，≥3条 → matched=false

请判断，回复JSON格式：
{"matched": true/false, "total_checked": 25, "monologue_leak_count": 3, "examples": [{"episode": 1, "scene": 2, "dialogue_excerpt": "问题台词摘要", "leak_type": "内心独白|信件全文|叙述旁白|情节概要", "note": "为什么判定为混入"}], "reason": "说明dialogue字段是否干净"}

## 无注水/拖沓段落 [basic]

### 评分标准（matched: true/false）
### true = 无明显注水，剧情紧凑
### false = 有明显注水/拖沓段落

**【本检查项只管一件事】**：剧本中是否有明显注水或拖沓的段落。
- ✅ 检查：是否有对推进剧情无贡献的冗余场景
- ✅ 检查：中段是否出现"塌腰"（前几集紧凑、中间几集突然拖沓）
- ❌ 不检查：节奏是否有起伏（那是 pacing 的范围）
- ❌ 不检查：台词是否啰嗦（那是 dialogue 检查项的范围）

**什么是"注水/拖沓"**：
短剧观众忍耐度极低，一旦感觉"这段没必要"就会弃剧。
注水 = 这个场景/这几集删掉不影响主线，纯粹是凑集数。

**典型注水模式**：
- **回忆杀注水**：大量篇幅重复回忆之前已经演过的场景
- **配角支线过度展开**：配角的感情线/日常占了好几集，跟主线无关
- **反复拉扯**：男女主吵架-和好-再吵架-再和好，同样模式循环3次以上
- **铺垫过长**：用了好几集来铺垫一个本可以1集讲完的前因
- **中段塌腰**：开头几集很紧凑，到中段突然节奏变慢、无重要事件发生

**不算注水**：
- 情感过渡段（主角在重大事件后需要消化，1-2个场景的过渡是合理的）
- 铺垫段（为后续反转做必要铺垫，虽然当前节奏慢但信息有用）
- 人物建立段（初期用1-2集建立人物关系和世界观是正常的）

**验证流程**：
1. 通读大纲或所有剧本，关注中段（总集数的30%-70%区间）
2. 标记可疑的注水场景/集
3. 对每个可疑段落判断：删掉它，主线剧情是否不受影响？
4. 如果是 → 注水；如果删掉会影响后续理解 → 不是注水
5. 无注水或注水段落≤2处且每处不超过1集 → matched=true
6. 注水段落≥3处，或连续2集以上注水 → matched=false

请判断，回复JSON格式：
{"matched": true/false, "filler_segments": [{"episodes": "15-17", "description": "内容概述", "impact_if_removed": "对主线无影响/有影响"}], "mid_section_assessment": "紧凑/正常/塌腰", "reason": "说明剧本是否有注水问题"}

## 氛围营造能力 [advanced]

### 评分标准（matched: true/false）
### true = 优秀（场景头信息和内容描述有效营造了符合剧情的氛围）
### false = 未达优秀标准（场景描述功能性但无氛围）

**【重要】本检查项的范围约束**：
- ✅ 检查：场景头信息（时间/地点/天气/光线/音效/atmosphere）是否服务于情绪表达
- ✅ 检查：环境细节是否营造了符合剧情需要的氛围
- ✅ 检查：音效/BGM标注是否与场景情绪匹配
- ❌ 不检查：画面是否可视觉化（那是basic层/视觉化维度的范围）
- ❌ 不检查：台词质量（那是dialogue_colloquial检查的范围）
- ❌ 不检查：情节逻辑（那是plot_logic检查的范围）

**什么是"氛围营造"**：
短剧的氛围不只靠情节，还靠场景的整体"感觉"——时间、光线、天气、音效、空间布局的组合，
在观众理性理解情节之前就先建立情绪基调。好的氛围让观众"还没看到角色说话就已经进入状态了"。

**未达标（matched: false）：场景无氛围**

**问题类型1：场景头信息是空模板**
- 场景头只有地点名，没有时间/天气/光线/音效等氛围信息
- 示例（无氛围）：
  ```json
  {"location": "办公室", "time": "白天"}
  ```
- 示例（有氛围）：
  ```json
  {"location": "顶楼办公室", "time": "黄昏", "lighting": "夕阳从落地窗斜射入，长长的阴影横过地板", "weather": "阴转多云", "sfx": "空调嗡鸣，远处有救护车鸣笛", "atmosphere": "压抑"}
  ```

**问题类型2：氛围与情绪不匹配**
- 有氛围描写但与场景情绪冲突
- 示例：紧张对峙场景设定在"阳光明媚的午后公园"且无任何压迫感的描写
- 示例：甜蜜告白场景设定在"阴暗潮湿的地下车库"且无任何浪漫氛围处理
- 核心判断：观众看到场景设定后，是否已经进入了"正确的情绪"？

**问题类型3：所有场景氛围雷同**
- 无论什么情节，场景头信息都差不多
- 示例：所有场景都是"白天/室内/无特殊"，没有利用环境变化来辅助叙事
- 好的剧本会让不同情绪段落的场景氛围有明显差异

**达标（matched: true）：氛围营造有效**

**有效氛围的特征**：
- **时间为情绪服务**：紧张→深夜/黄昏；甜蜜→午后阳光/傍晚；悲伤→雨天/清晨
- **光线为情绪服务**：对峙→逆光/阴影；浪漫→暖光/柔光；悬疑→忽明忽暗/冷光
- **音效/BGM标注**：关键场景有音效或BGM提示（哪怕是简单的"安静"也是一种音效设计）
- **环境细节**：不只是"在办公室"，而是"杯子里的咖啡凉了"、"窗外的雨打在玻璃上"
- **差异化**：不同情绪段落的氛围明显不同——观众可以通过场景设定就预感到接下来的基调

**各类型场景的优秀氛围示例**：

**紧张/对峙**：
```
深夜，密闭会议室。只有桌上台灯亮着，两人的脸半明半暗。
空调已关闭，空气闷热。远处有钟表滴答声。
```

**甜蜜/浪漫**：
```
初秋傍晚，街边的银杏叶刚开始变黄。夕阳从建筑缝隙间照进来，
把两人的影子拉得很长。背景有隐约的吉他弹唱声。
```

**悲伤/离别**：
```
医院走廊尽头，白色日光灯闪烁不定。消毒水的味道弥漫。
广播播报着上一个患者的名字。窗外是灰蒙蒙的雨。
```

**悬疑/惊悚**（短剧常见）：
```
凌晨三点，老旧小区楼道。声控灯忽亮忽灭，墙上的小广告被风吹得沙沙响。
远处传来猫叫。楼梯间有不明来源的水渍。
```

**验证要点**：
1. **场景头信息检查**：
   - 抽查5-8个场景的头信息（scene_header）
   - 是否有时间/光线/天气/音效/atmosphere等氛围元素？
   - 这些元素是否服务于场景情绪？
2. **氛围-情绪匹配度**：
   - 对峙场景的氛围是否压迫？
   - 甜蜜场景的氛围是否温暖？
   - 悬疑场景的氛围是否诡异？
3. **差异化**：
   - 不同情绪段落的场景氛围是否有变化？
   - 全部场景的氛围设定是否雷同？
4. **音效/BGM**：
   - 关键场景是否有音效/BGM标注？
   - 标注是否匹配场景情绪？

**判断流程**：
1. 抽样检查不同情绪类型的场景（5-8个）
2. 评估每个场景的氛围元素丰富度和情绪匹配度
3. 检查场景间的差异化
4. **两步判断**：
   - **核心条件**（必须满足）：抽查场景中 **≥60%** 有有效氛围元素（不是空模板）且与情绪基本匹配
   - **加分条件**（不强制）：不同情绪段落的氛围有明显差异化
5. 核心条件满足 → matched=true（差异化作为 reason 中的评价，不一票否决）
6. 核心条件不满足（多数场景空模板或不匹配）→ matched=false

**⚠️ 关键容错**：
- 不要求每个场景都有完美氛围——整体有意识地在做氛围营造即可
- 差异化是锦上添花，不是一票否决条件。模型如果在多数场景写了丰富且匹配的氛围，即使差异化一般也应 pass
- 短剧场景多（30-80集×多场景），不能因为少数场景氛围薄弱就 fail
